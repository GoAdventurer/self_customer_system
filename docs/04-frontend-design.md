# 企业级智能客服系统 · 前端设计规范

> 设计美学: **"Nordic Precision"** — 北欧极简精密感 × 数据智能可视化  
> 核心理念: 信息密度与呼吸感的精确平衡，让复杂系统如手术刀般清晰

---

## 1. 设计总览

### 1.1 界面矩阵

| 界面 | 使用者 | 核心场景 | 信息密度 | 审美基调 |
|------|--------|---------|---------|---------|
| **用户对话界面** | 终端用户(C端) | 咨询/退款/投诉 | 低(聚焦对话) | 轻盈、亲切、信任感 |
| **坐席工作台** | 客服坐席(B端) | 人机协同/高效处理 | 高(多面板) | 专业、高效、辅助感 |
| **运营监控大盘** | 技术/业务负责人 | 监控/决策/分析 | 极高(数据密集) | 精密、冷静、掌控感 |

**核心体验**: 如同与一位耐心、专业的人类客服对话，而非"和机器交互"。  
**审美方向**: 浅色主题、圆润气泡、柔和动画、温暖中性色、适度留白。

### 3.3 关键交互设计

| 交互场景 | 设计方案 |
|---------|---------|
| AI正在回复 | 渐入气泡+三点呼吸动画,非空白等待 |
| 富媒体卡片 | 订单/物流/退款进度嵌入对话流,可展开 |
| 文件上传 | 拖拽区+进度条+解析状态(解析中→完成) |
| 排队等候 | 顶部进度条+预估时间+排位,可选"留言回拨" |
| 转人工过渡 | 柔和动画切换头像/状态标签,上下文无缝 |
| 情绪感知 | 不显式告知用户(避免不适),但调整回复语气 |
| 满意度评价 | 会话结束后底部浮出,五星+快捷标签+文字框 |

### 3.4 响应式适配

- **Mobile(375px)**: 全屏对话,底部输入栏固定,气泡宽度≤80%
- **Tablet(768px)**: 居中对话区(max-width 640px),两侧留白
- **Desktop(1280px)**: 右侧可浮出FAQ面板/订单面板(可关闭)

### 4.3 关键组件设计

**AI建议卡片**（最核心的人机协同组件）:
- 紫色渐变左边框标识"AI生成"
- 置信度指示器(高/中/低 三色)
- 操作按钮: 一键发送 | 编辑后发送 | 忽略
- 编辑时高亮AI原文与人工修改的diff

**情绪曲线**:
- 迷你折线图(sparkline),颜色渐变(绿→黄→红)
- 当前轮次情绪标签+趋势箭头
- 触发升级时闪烁警告

**会话列表项**:
- 左侧彩色圆点(红/黄/绿对应紧急度)
- 主内容: 用户名(脱敏) + 问题摘要
- 副信息: 等待时长 + AI/人工状态
- 未读消息badge

### 5.3 数据可视化规范

| 数据类型 | 可视化形式 | 设计规则 |
|---------|-----------|---------|
| 核心KPI | 大号数字 + 趋势箭头 + 环比百分比 | 数字用mono字体,趋势色(绿涨红跌) |

### 5.4 实时更新策略

- **KPI数字**: WebSocket推送,5s刷新一次,数字滚动动画
- **图表数据**: 30s增量更新,新数据点从右侧平滑滑入
- **告警**: 实时推送,新告警从顶部滑入+轻微震动
- **系统级别**: 变化时全屏瞬闪对应颜色(0.3s)+音效

## 6. 组件设计规范

### 6.1 按钮体系

| 类型 | 使用场景 | 视觉 |
|------|---------|------|
| Primary | 主要操作(发送/确认/提交) | 填充accent色,白字,hover亮20% |
| Secondary | 次要操作(取消/编辑) | 描边+透明底,hover填充10%opacity |
| Ghost | 辅助操作(展开/收起) | 无边框,hover底色变化 |
| Danger | 危险操作(删除/拒绝) | 红色填充/描边 |
| AI Action | AI建议操作 | 紫色渐变描边+微光动画 |

### 6.2 卡片体系

- **信息卡片**: 圆角10px, 1px border(微透明白), 内间距24px
- **数据卡片**: 圆角10px, 无border用阴影区分, 左侧彩色指示条
- **AI建议卡片**: 左侧渐变紫色边框(3px), 底色微紫(#6366F1的5%opacity)
- **告警卡片**: 左侧对应告警色(3px), 行高紧凑, hover展开详情

### 6.3 图标体系

- 使用 Lucide Icons (线性、几何、一致性强)
- 尺寸: 16px(行内) / 20px(按钮/列表) / 24px(导航)
- 颜色跟随文字层级(primary/secondary/tertiary)
- 状态图标使用填充圆点(●)而非线性图标,提升识别效率

## 7. 交互模式

### 7.1 AI 内容标识规范

所有 AI 生成/辅助的内容必须有明确视觉区分:

```
┌─ AI生成内容标识方式 ──────────────────────────────┐
│                                                   │
│  对话气泡:    🤖 图标 + "AI助手" 标签             │
│  建议回复:    紫色左边框 + "AI建议" 标签           │
│  自动填充:    紫色虚线底线 + 淡紫底色              │
│  摘要/分析:   "由AI生成" 角标 + 置信度指示         │
│                                                   │
│  原则: 用户始终知道哪些是AI、哪些是人工            │
│                                                   │
└───────────────────────────────────────────────────┘
```

### 7.2 加载状态

- **骨架屏(Skeleton)**: 首次加载使用,保持布局稳定
- **脉冲点(Pulse)**: AI正在思考/生成,三点呼吸动画
- **进度条**: 文件上传/解析,带百分比
- **数字过渡**: 数据刷新时数字平滑过渡(非跳变)

### 7.3 错误处理

- **网络断开**: 顶部红色banner + 自动重连倒计时
- **操作失败**: toast通知(右上角,5s自动消失) + 重试按钮
- **AI不可用**: 平滑切换为模板回复模式,不中断服务

## 8. 无障碍(Accessibility)

- 所有交互元素符合 WCAG 2.1 AA 对比度要求
- 键盘可完成所有核心操作(focus可见、tab顺序合理)
- 图表提供数据表格替代视图
- 颜色不作为唯一信息载体(配合图标/文字)
- 屏幕阅读器适配(aria标签、语义化HTML)

## 9. 技术选型建议

| 层面 | 推荐方案 | 理由 |
|------|---------|------|
| 框架 | React 18+ (Next.js) | 生态成熟、SSR支持、团队熟悉度 |
| 状态管理 | Zustand + React Query | 轻量、服务端状态分离 |
| 样式方案 | Tailwind CSS + CSS Variables | 原子化高效、主题切换方便 |
| 图表 | ECharts (大盘) + Recharts (轻量) | ECharts大数据量性能好 |
| 动画 | Framer Motion | React生态最佳、声明式API |
| 实时通信 | WebSocket + EventSource(SSE) | 对话用WS双向、指标用SSE单向 |
| 图标 | Lucide React | 轻量、树摇动、风格一致 |
| 组件库 | 自建(基于Radix UI primitives) | 无样式基础+完全自定义设计 |
