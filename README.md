# NexusAI · 企业级智能客服系统

> 面向大型电商企业的全渠道、多租户、人机协同智能客服系统。  
> 本仓库为**架构设计方案**，包含完整的系统设计文档与前端原型。

---

## 系统定位

解决大型电商客服场景的核心矛盾：**高并发 × 长尾业务 × 强合规 × 成本压力**。

| 目标 | 指标 |
|------|------|
| 降本 | AI 自助解决率 ≥ 70%，单会话成本 < ¥1 |
| 提效 | 平均首响 < 3s，解决时长降低 40% |
| 提质 | 意图准确率 ≥ 92%，关键流程零误操作 |
| 可控 | 全链路可观测、可审计、可回溯 |

## 设计原则

1. **分层解耦** — 每层职责单一，可独立演进与扩缩容
2. **编排优先** — 跨系统副作用走显式编排 + 幂等 + 补偿
3. **混合智能** — 规则/分类器/检索/大模型按"够用最低成本"分工
4. **人机协同** — 模型理解与起草，高风险由人确认或确定性兜底
5. **可观测可治理** — 全链路埋点、可灰度、可回滚、可审计

## 技术栈（参考）

| 层面 | 选型 |
|------|------|
| 编排引擎 | LangGraph (任务图/状态机) |
| 部署 | K8s + HPA 弹性伸缩 |
| 模型 | 本地私有化(敏感) + 云端大模型(复杂) |
| 知识库 | Milvus/PGVector + ES + BM25 混合检索 |
| 缓存 | Redis (会话状态) + 语义缓存 (向量相似) |
| 前端 | React 18 + Tailwind + Framer Motion |
| 可观测 | OpenTelemetry + 时序库 + 自建大盘 |

## 演进路线

| 阶段 | 范围 | 关键里程碑 |
|------|------|-----------|
| **MVP** | 单租户 · 文本渠道 · FAQ+工单+转人工 | 规则+小模型+基础RAG |
| **增强** | 多渠道 · 复杂流程 · 人机协同工作台 | 模型路由+缓存+Saga |
| **规模化** | 多租户 · 数据飞轮 · 全面可观测 | 成本治理+合规体系化 |
